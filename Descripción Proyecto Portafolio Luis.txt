EL TEMA DEL PORTAFOLIO

  Al principio no estaba muy claro qué hacer para el portafolio, pero después de pensar un poco, decidí crear una galería de paisajes. La idea es reunir varias imágenes 
de lugares naturales, mostrando diferentes tipos de paisajes. Así, el portafolio tiene un enfoque visual, sencillo y atractivo, pero también permite practicar un poco 
con el diseño web, creando un espacio limpio y ordenado donde se puedan ver las imágenes de manera clara y agradable.

--------------------------------------------------------------------------------------------------------------------------------------------------------------------------

PUNTOS A FAVOR

1. Mi experiencia creando el portafolio

  Para hacer este portafolio, me basé en algunos ejemplos de YouTube, tutoriales de HTML y CSS que encontré en varios sitios como w3schools, 
además de lo que aprendí en clase. Gracias a eso, pude organizar mejor la página y darle un toque más profesional al diseño. Desde el principio,
quería que el portafolio se viera moderno y bien ordenado, así que usé algunas técnicas como gradientes, flexbox y grids. La mayoría de estas 
ideas las saqué de referencias en línea y las fui adaptando a mi gusto para que el contenido se viera de la mejor forma posible.

2. Cómo estructuré el CSS y HTML

  Empecé por configurar los estilos generales para que la página se viera bien en diferentes navegadores. Luego, diseñé el header, donde puse el logo y 
los enlaces de navegación. Usé Flexbox porque da mayor facilidad a la hora de alinear los elementos de manera ordenada y que se adapten a diferentes pantallas. 
También le puse un degradado verde y bordes redondeados para que se vea más profesional. Para hacerlo, tomé algunas ideas de tutoriales en YouTube y W3Schools, 
donde enseñan cómo hacer encabezados modernos con flexbox y gradientes.

3. Sección de Introducción

  Para la introducción, quería que tuviera un aspecto limpio y elegante, por lo que usé un gradiente de fondo, una sombra (box-shadow) para resaltar la sección
y una fuente más llamativa para el título. Además, el texto está justificado para que tenga una mejor lectura. Aquí seguí una estructura similar a ejemplos de 
páginas de presentación que encontré en tutoriales.

4. Galería de Imágenes

  Sabía que quería una galería de imágenes bien organizada, así que decidí usar CSS Grid en lugar de Flexbox, ya que Grid me dio más control sobre el diseño. 
También agregué un efecto hover para que las imágenes se agrandaran un poco al pasar el cursor, algo que vi en varios tutoriales y me pareció una buena idea para
hacerlo más dinámico.

5. Sección Final y Footer

  Para cerrar el portafolio, diseñé una última sección con un mensaje final y un footer con un fondo oscuro y un borde superior, algo que vi recomendado en varias guías
de diseño web.

6. ¿Por qué mezclé inglés y español en el código?

  Desde el principio, decidí usar nombres de clases en inglés porque en desarrollo web es una práctica común y la mayoría de los tutoriales que seguí lo hacían así. 
Además, esto hace que el código sea más claro por si en el futuro necesito hacer modificaciones o compartirlo con alguien más. Sin embargo, el contenido dentro de la
página está en español, ya que es el idioma en el que quiero presentar la información.

INCONVENIENTES

1. Función de Galería para Ampliar Imágenes: No pude implementar correctamente la función para ampliar las imágenes al hacer clic (modal). Hubo problemas al intentar 
integrar el HTML, CSS y JavaScript (de este último tengo poco conocimiento). Hice varios intentos por vincular los archivos y ajustar el código incluso usando como 
guía w3schools, no pude hacer que funcionara. 

2. Interacción con las Imágenes: Aunque puse algunos efectos bonitos en las imágenes, como el cambio de tamaño al pasar el ratón sobre ellas, no pude hacer que al hacer clic
en una imagen se ampliara, como había planeado. 

3. Comportamiento del Modal: Utilicé un código basado en la página de W3Schools para implementar la ventana emergente (modal) con la imagen ampliada, pero no logré que funcionara.
Aunque el modal estaba en el código, no lograba mostrar la imagen como esperaba al hacer clic en ella.

CONCLUSIÓN FINAL: Aunque intenté agregar varias funcionalidades y estilos en la página, algunas cosas no salieron como esperaba. Como no pude hacer que ciertas características, 
como la galería para ampliar imágenes, funcionaran a tiempo, decidí quitar esas partes del código final. Así, el proyecto quedó más limpio y funcional, sin cosas que no estaban 
funcionando bien. Espero poder seguir aprendiendo y mejorar estos detalles en futuros proyectos.
